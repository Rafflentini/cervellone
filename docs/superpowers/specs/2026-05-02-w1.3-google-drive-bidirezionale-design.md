## Componenti

### 1. `src/lib/drive.ts` (esteso)

Funzioni esistenti (10): `listFiles`, `searchFiles` (name only), `createFolder`, `moveFile`, `renameFile`, `readDocument` (Google Docs only), `createDocument`, `readSheet`, `writeSheet`, `appendSheet`.

**Nuove funzioni:**
- `searchFilesFullText(query, folderId?)` — query Drive con `fullText contains` per cercare nel CONTENUTO
- `downloadFile(fileId)` — download binario, ritorna Buffer
- `readPdfFromDrive(fileId)` — `downloadFile` + `pdf-parse` → testo
- `readDocxFromDrive(fileId)` — `downloadFile` + `mammoth` → testo
- `readXlsxFromDrive(fileId)` — `downloadFile` + `JSZip` parse XML → righe
- `findCantiereFolder(query)` — legge REGISTRO_CANTIERI, fuzzy match per nome cliente/cantiere → folder ID
- `findClienteFolder(query)` — stesso pattern per Studio Tecnico ATTIVI

  recentHistory: string,

  // 1. Estrai cantiere/cliente da prompt + history (regex su pattern noti)
  // 2. Lookup REGISTRO_CANTIERI o REGISTRO_PROGETTI Sheet
  // 3. Se match → folder ID + path
  // 4. Se no match → fallback per tipo doc, isFallback=true

### 4. Hook auto-save in `route.ts` Telegram

Dopo `parseDocumentBlocks(fullResponse)`, per ogni block `type === 'document'`:
```ts
const docType = inferDocumentType(block.content, userText)
const result = await saveDocumentToDrive(
  block.content, title, docType, userText, recentHistoryText
).catch((err) => {
  console.error('Drive save failed:', err)
  return null
})
if (result) {
  textParts.push(`📂 Salvato in *${result.folderPath}*\n👉 ${result.driveUrl}`)
}
```

Stesso hook in `src/app/api/chat/route.ts` (web) e `trigger/cervellone-long-task.ts`.

## Mappatura cartelle (Opzione Y + fallback X)

| Tipo doc | Cartella primaria (Y) | Fallback (X) |
|----------|----------------------|--------------|
| POS | `/CANTIERI ATTIVI/[cantiere]/POS/` | `/POS/` |
| Preventivo | `/Studio Tecnico ATTIVI/[cliente]/Preventivi/` | `/Studio Tecnico ATTIVI/_Bozze/` |
| Perizia | `/Studio Tecnico ATTIVI/[cliente]/Perizie/` | `/Studio Tecnico ATTIVI/_Bozze/` |
| CME | `/Studio Tecnico ATTIVI/[cliente]/Computi/` | `/Studio Tecnico ATTIVI/_Bozze/` |
| Relazione tecnica | `/Studio Tecnico ATTIVI/[cliente]/Relazioni/` | `/Studio Tecnico ATTIVI/_Bozze/` |

| Generico | — | `/Studio Tecnico ATTIVI/_Bozze/` |

**Bot dichiara sempre il path scelto** nel messaggio finale Telegram/web. Se sbaglia, l'utente corregge in chat ("no, mettilo in `/Cantieri/Rossi/`") → Cervellone esegue `drive_move_file`.

## Risk e mitigations

| Risk | Severità | Mitigazione |
|------|----------|-------------|
| Service Account compromesso → accesso totale Drive | 🔴 ALTO | Condividere SOLO le 9 cartelle specifiche, NON Drive root. Chiave JSON solo su Vercel env Sensitive. Rotazione chiave annuale (calendario). |
| REGISTRO_CANTIERI non aggiornato → wrong folder | 🟡 MEDIO | Fallback X automatico + bot dichiara path → utente corregge se sbagliato |
| Quota Drive API (10k req/100s) | 🟢 OK | Single-user, lontani dal limite |
| File binari grandi (>50MB) timeout | 🟡 MEDIO | Skip file > 20MB con warning, leggi solo i primi N KB di PDF lunghi |
| GOOGLE_SERVICE_ACCOUNT_KEY non Sensitive in Dev env (limit Vercel) | 🟢 OK | Accept rischio (team Restruktura = solo Raffaele), documentato in feedback |
